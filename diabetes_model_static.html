Diabetes Screening Model

Overview

Millions of individuals live with undiagnosed type 2 diabetes, accumulating years of damage that increases their risk of cardiovascular disease, kidney failure, nerve damage, and vision loss—complications that early detection could otherwise prevent or delay. Yet, the populations who would benefit most from early screening — like those in under-resourced communities, rural areas and developing nations — often lack the access, funding and infrastructure required for comprehensive diagnostic testing.

A lightweight, low-cost and noninvasive screening model may be beneficial in helping to address this disparity; by equipping and enabling frontline healthcare workers, community health programs and healthcare practitioners to rapidly screen patients, we may be able to extend the reach of diabetes detection efforts to population health groups that need it most.

What's captured here is, of course, not intended to be a diagnostic tool, but rather a scalable and high-volume ‘first filter’ to help stratify risk and guide downstream efforts such as referrals, education programs or follow-up testing. Consider this a 'minimum viable product' that establishes a conceptual framework for future iteration, development and research.

Variables

This model uses 9 easily obtained clinical characteristics and applies a decision tree algorithm to generate a risk classification of either 'Diabetic' or 'Normal'. These variables may be sourced from patient charts or during the intake process. As depicted further below, building out this algorithm programatically is easy and could be deployed in several capacities including Clinical Decision Support tools, integration with Electronic Health Record modules, business intelligence dashboards and data marts, or even just ad-hoc as a standlone application using patient sets. These variables were sourced and selected based on a study published here.

Functional Goals

  • Improve clinical outcomes through earlier risk detection and intervention
  • Improve productivity of health care workers by way of resource reallocation - i.e. physician involvement in low-risk screening can be reduced or delegated
  • Reduce lab utilization by right-sizing frequency of blood glucose testing among certain low-risk patient cohorts
  • Improve accuracy and time to clinical diagnosis through broader screening
  • Drive greater patient involvement in their own health through earlier educational and lifestyle intervention

    First, can define a resuable function that reads in a dictionary containing our 9 variables

    
def assess_diabetes_risk(patient_data):
"""
Assess diabetes risk based on patient characteristics

Args:
	patient_data (dict): Dictionary containing patient information with keys:
		- age: int
		- family_history_diabetes: bool
		- bmi: float
		- prefers_salty_food: bool
		- physically_active: bool
		- history_cvd_stroke: bool
		- work_stress: str ('HIGH' or 'LOW/MODERATE')
		- has_hypertension: bool
		- sex: str ('MALE' or 'FEMALE')

Returns:
	str: 'NORMAL' or 'DIABETIC'
"""

age = patient_data['age']
family_history_diabetes = patient_data['family_history_diabetes']
bmi = patient_data['bmi']
prefers_salty_food = patient_data['prefers_salty_food']
physically_active = patient_data['physically_active']
history_cvd_stroke = patient_data['history_cvd_stroke']
work_stress = patient_data['work_stress']
has_hypertension = patient_data['has_hypertension']
sex = patient_data['sex']

if age <= 49:
	if family_history_diabetes:
		if bmi <= 25:
			return 'NORMAL'
		else:  
			return 'DIABETIC'
	else:  
		if bmi <= 25:
			return 'NORMAL'
		else:  
			if prefers_salty_food:
				if age <= 34:
					return 'NORMAL'
				else:  
					if physically_active:
						if history_cvd_stroke:
							return 'DIABETIC'
						else:  
							return 'NORMAL'
					else:  
						return 'DIABETIC'
			else:  
				return 'NORMAL'

else:  
	if work_stress == 'HIGH':
		if family_history_diabetes:
			return 'DIABETIC'
		else:  
			if bmi <= 25:
				return 'NORMAL'
			else:  
				return 'DIABETIC'
	else:  
		if bmi <= 25:
			if prefers_salty_food:
				if has_hypertension:
					return 'DIABETIC'
				else:  
					if sex == 'MALE':
						if history_cvd_stroke:
							return 'DIABETIC'
						else:  
							return 'NORMAL'
					else:  
						return 'NORMAL'
			else:  
				return 'NORMAL'
		else:  
			if family_history_diabetes:
				return 'DIABETIC'
			else:  
				if prefers_salty_food:
					return 'DIABETIC'
				else:  
					return 'NORMAL'

    Now, we're ready to invoke the function using our test data from above.

    
def patient_assessment():

import pandas as pd

demo_patients = create_demo_patients()


results = []
for patient in demo_patients:
	risk_assessment = assess_diabetes_risk(patient)
	patient_summary = {
		'Name': patient['name'],
		'Age': patient['age'],
		'BMI': patient['bmi'],
		'Family_History': patient['family_history_diabetes'],
		'High_Stress': patient['work_stress'] == 'HIGH',
		'Physically_Active': patient['physically_active'],
		'Risk_Assessment': risk_assessment
	}
	results.append(patient_summary)


df = pd.DataFrame(results)

print("=== DIABETES RISK ASSESSMENT ===\n")
print(df.to_string(index=False))

print(f"\n=== SUMMARY ===")
print(f"Total patients assessed: {len(results)}")
print(f"Patients flagged as DIABETIC risk: {sum(1 for r in results if r['Risk_Assessment'] == 'DIABETIC')}")
print(f"Patients classified as NORMAL: {sum(1 for r in results if r['Risk_Assessment'] == 'NORMAL')}")

return df

# Invoke Function 
if __name__ == "__main__":
scored_df = patient_assessment()

    And, finally see the results as calculated by the screening algorithm. While there's nothing remarkable about this prototype in a vacuum, it's easy to extrapolate how this type of framework may eventually scale its way into clinical workflows. Imagine a swath of approved and governed models that sit behind-the-scenes, constantly conducting 'chart review' across the totality of the Electronic Health System patient population. Algorithms could very feasibly monitor, aggregate and evalute everything from labs, images and clinical notes to referral summaries, patient-reported outcomes and SDOH questionnaires. This level of augmentation could significantly increase physician capacity and patient outcomes.
